1.1 Notes: The Basics of Web and HTML

Components of the Web

The web consists of the following:
  • You - none of this matters without people using the web.
  • Computer and browser - for example, I'm typing this in a window in the Chrome browser on my MacBook Pro laptop.
  • The Internet - a global system of computers that use a protocol called HTTP to communicate. (I'm getting ahead of myself.)
  • HTTP - hypertext transfer protocol, the protocol that is the foundation of the web.
  • Servers - computers optimized for hosting files.
How does it all work together?

You open a browser (like Chrome) on your laptop (like I said, I use a Macbook Pro). In the address bar, you enter a URL (oops, should have mentioned that one earlier - more info here) that tells your computer what server (like google.com) to access. HTTP is the protocol Chrome uses to send requests to - and receives responses from - the server.

Tags and Elements

One of the first things we learned about is tags and elememts. A tag is a hidden keyword that tells the browser how to format content. An element consists of an opening tag, content, and a closing tag.

Some example are the bold tag, b, and the emphasis tag, em.

There are other types of tags that don't need to be closed. The br tag adds a line break. The img tag is used to embed images, like I added above. The a tag is used to add links.

Inline vs Block

Some tags like b, em, and even br are inline tags, which means they are normally displayed without line breaks.

Other tags like p and div, are block tags. That means they are their own elements and can have a specified height and width.

1.2 Notes: Creating a Structured Document

Overview

We're starting with HTML and CSS, focusing on the structure of HTML documents and how browsers interpret structure. This is a great way to start because front end web development is inherently visual, so changes are automatically visible, and it's really easy to share a website with someone.

The Basics
  • HTML is the structure of the web page.
  • CSS styles the page.
  • JS adds interactive components. (We're not doing any JS in this Nanodegree.)
  • DOM (Document Object Model) refers to the tree-like structure of the page.
How Browsers Interpret Code

Chrome's Developer Tools shows how the browser interprets code. (To reach Dev Tools, click the hamburger menu, more tools, and then developer tools.) There are a few things to remember about the code:
  • Not all elements are visible.
  • All elements are rectangular, even if they don't appear to be!
  • If you go deep enough in the structure, you can read the text on the page.
Breaking Down a Design / Boxifying Design

Everything in a web page is a box. When breaking down a design, there are a few different ways to "boxify" the design:
  • Draw boxes on with a graphics program.
  • Doodle on a piece of paper.
  • Print the page and draw on it.
  • Print the page and cut it out with scissors
Especially for beginners, doodling on paper seems like a good way to go. Start with the largest boxes, working from big to small. The basic layout of the boxes is most important for the time being - positioning and content can be added in the future.

From Boxes to HTML

How should we create the basic structure in HTML? Create a div for every large box, and give it a name. The name we give the div is called the class. The class should give you an idea of what it's used for (like "title" or "header"), and each class name should make sense so you can remember what it's supposed to do.

1.3 Notes: Adding CSS Style to HTML Structure

Overview

In this lesson we're diving further into CSS and learning the beginnings of an important step in learning how to apply CSS in various ways in order to avoid repetition.

Notes on HTML Formatting

Up to this point we've just been working with the body of our HTML documents and ignoring the rest, for the most part. Here are a few things we've glossed over up to this point.
  • <!DOCTYPE html> - this goes at the very top of the HTML document and tells the browser it's looking at an HTML file.
  • <html></html> - everything on the page should be enclosed within these tags.
  • <head></head> - things like metadata, a link to the CSS file, and the page title should be in this section of the document.
  • <body></body> - and this is where the body of the document goes.

CSS Basics

CSS stands for Cascading Style Sheets. The most important part of this is the C, which stands for cascading. The concept of cascading means that, as you move through the document tree, style properties are inherited from the enclosing element, but more the most specific rule applies. There's a nice section on inheritance on the Wikipedia page for CSS - it definitely helped clear things up. In general, text styles are inherited, and box styles are not.

Where does CSS live?

CSS information can be stored in two places:
  • In the head section, within <style></style> tags. This usually isn't a great idea for anything other than small projects - the whole point is to write one CSS document and refer back to it.
  • As a separate, linked document. This is almost always the best way to go.
Here's how it should look when linking the CSS file.
<head>
     <link rel="stylesheet" type="text/css" href="main.css">
</head>

CSS Selectors and Reference

Style is applied to a particular selector. There are lots of different types of selectors, but for starting out we can stick with two: tags and classes. There's a lot to CSS, and we're only scratching the surface in this lesson. Here are two great resources for more info:
  • Selectors
  • CSS reference
Box, Box, Box, Box, Box, Box, EVERYBODY!

We need to have a talk. About those boxes. There's much more to them than we've let on. They have four different parts:
  1. Margin - clears an area around the border; no color / completely transparent.
  2. Border - goes around padding and content; inherited from the color property of the box.
  3. Padding - clears an area around the content; affected by the background color of the box (the space inside the box that protects content directly).
  4. Content - the image or text that's appearing on the web page; other layers serve to protect it so they don't overlap or sit too close.
All of these properties can be individually tweaked using Chrome's developer tools. :)

Here's a nice visual representation of the different parts of the box from W3 Schools.


With all these different parts of the box, it gets tricky to calculate the overall size of any box. There are a few things we can do to simplify this:
  • Use the box-sizing: border-box property to make the element size include the border and padding - that way the size of the box remains constant.
  • Set the width in percentage, and the box will change based on screen size.

Box Positioning

Divs normally stack on top of one another (because they're block elements). CSS offers lots of options to change a page's layout, but that gets really complicated so we're just looking at one way to manage divs... we'll just look at one layout property in this lesson.

The property display: flex will allow divs to display side by side and distribute space within a container. The CSS Tricks website offers more info here.

Process and Tools

When we first look at a design mock, here's how we should break it down (in case you forgot what we talked about in the last lesson.)
  1. Look for natural boxes.
  2. Look for repeated styles and semantic elements.
  3. Write your HTML.
  4. Apply styles, from biggest to smallest.
  5. Fix things.
Without explicitly discussing it, we've been following a process for this entire lesson:
  1. Write code.
  2. See how it looks vs the design mock.
  3. Made revisions.
  4. Repeat until it looks like the design we've been given.
